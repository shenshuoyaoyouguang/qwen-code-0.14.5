## How to Use These Guidelines

1. **First time in project**: Read `guides/` first for project-wide thinking
2. **Starting a task**: Check relevant domain directory for specifics
3. **When unsure**: Check `guides/` for common patterns

---

## Adding New Guidelines

New guideline files should:

- Be written in Markdown (`.md`)
- Follow the naming convention: `kebab-case-topic.md`
- Include a header with scope and purpose
- Be listed in the parent `index.md`

---

**Language**: All documentation should be written in English for AI compatibility.
